# App Taxonomy and Intent Dataset

Author: Maggie Ju  
Course: PSYCH 196B – Research in Cognitive Science

## Project Overview

This project organizes common mobile applications into a taxonomy based on product type and user intent. The goal was to better understand patterns in how different types of applications structure their onboarding experiences, general user flows, and unique features.

## Developing the Taxonomy

The taxonomy was developed by surveying commonly used digital products across multiple domains. Anmol and I first identified broad categories of applications based on their primary function and user goals (e.g., social media, e-commerce, productivity tools, streaming services, finance apps, etc.).

Next, I examined onboarding and feature screens across these products to identify recurring interaction patterns and intents. Apps were then grouped according to both their product type (e.g., messaging, marketplace, developer tools) and the user intent represented by their onboarding flows (e.g., account creation, discovery, search experience, etc).

The taxonomy was iteratively refined by comparing apps within each category and ensuring that the groupings captured meaningful similarities in UX/UI design.

## Screenshot Collection Process

Screenshots were collected from publicly accessible sources including my own application, app store previews, and internet search. The goal was to capture representative screens from the onboarding or recurring user experience of each product.

For each application, I documented key screens such as:

- onboarding and account setup screens
- primary features and core interactions
- search or discovery interfaces

These screenshots were then organized according to the taxonomy categories to allow for comparison across different types of applications. This taxonomy provides a structured way to analyze how different product categories guide user behavior through interface design and onboarding strategies.
