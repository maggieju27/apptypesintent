# App Taxonomy and Intent Dataset

Author: Maggie Ju  
Course: PSYCH 196B – Research in Cognitive Science

## Project Overview

This project organizes common mobile applications into a taxonomy based on product type and user intent. The goal was to better understand patterns in how different types of applications structure their onboarding experiences, general user flows, and unique features.

## Developing the Taxonomy

The taxonomy was developed by surveying commonly used digital products across multiple domains. Anmol and I first identified broad categories of applications based on their primary function and user goals (e.g., social media, e-commerce, productivity tools, streaming services, finance apps, etc.).

Next, I examined onboarding and feature screens across these products to identify recurring interaction patterns and intents. Apps were then grouped according to both their product type (e.g., messaging, marketplace, developer tools) and the user intent represented by their onboarding flows (e.g., account creation, discovery, search experience, etc).

The taxonomy was iteratively refined by comparing apps within each category and ensuring that the groupings captured meaningful similarities in UX/UI design.

## Step 1: Identify App Type Categories
I first grouped applications based on their primary user goal and product function. For example:

- Social Media and Messaging → communication and interaction  
- E-commerce, Marketplace, Food Delivery → browsing and transactions  
- Streaming Media and News → content consumption and discovery  
- Finance → monitoring and transactions  
- Productivity and Developer Tools → creation, editing, and system control  

This grouping reflects how UX research commonly categorizes products by user motivation and system purpose, which strongly influence interface design and interaction patterns. Research from the Interaction Design Foundation and Nielsen Norman Group shows that users form mental models specific to product domains, meaning that apps within the same category tend to share navigation structures, interaction patterns, and expectations.

### Step 2: Identify Intent Categories

Next, I analyzed screenshots across apps to identify recurring user goals represented in interface screens. These became the intent categories. The selected intents represent core interaction types that appear across most digital systems:

- Onboarding, Configure, System Feedback → align with usability principles such as learnability, system status visibility, and user control  
- Browse, Search, Discover → represent different modes of information seeking:
  - Search = goal-directed retrieval  
  - Browse = open-ended scanning  
  - Discover = system-driven recommendations  
- Detail / View → focused inspection of content or items  
- Create, Edit, Interact → content production and engagement  
- Transact → payment, booking, or conversion moments  
- Monitor → tracking progress, status, or analytics  

These categories are supported by major design systems and UX research, including:

- Apple Human Interface Guidelines (navigation, feedback, onboarding, system status)
- Google Material Design (actions, states, feedback, and structure)
- Nielsen Norman Group usability heuristics (visibility of system status, user control, recognition over recall)

These sources consistently frame interfaces as systems built around user goals, feedback loops, and task completion, which directly informed the intent taxonomy.

### Step 3: Cross-Product Comparison and Refinement
After defining initial categories, I compared screenshots across different apps to test whether the taxonomy:

- captured meaningful differences in user intent  
- avoided overlapping or redundant categories  
- could be applied consistently across multiple product types  

For example:
- *Browse* and *Search* were separated because they reflect different cognitive behaviors  
- *Discover* was separated from *Browse* to capture algorithm-driven exploration  
- *Detail / View* was separated from discovery to represent focused inspection  

This step ensured that each category reflects a distinct interaction pattern, rather than just a visual difference.

### Step 4: Final Consolidation

Finally, I refined category definitions to ensure consistency across the dataset. When a screen could fit multiple categories, it was assigned based on its primary user intent. This approach prioritizes analytical clarity and enables meaningful comparison across products.

## Screenshot Collection Process

Screenshots were collected from publicly accessible sources including my own application, app store previews, and internet search. The goal was to capture representative screens from the onboarding or recurring user experience of each product.

For each application, I documented key screens such as:

- onboarding and account setup screens
- primary features and core interactions
- search or discovery interfaces

These screenshots were then organized according to the taxonomy categories to allow for comparison across different types of applications. This taxonomy provides a structured way to analyze how different product categories guide user behavior through interface design and onboarding strategies.
